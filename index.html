06 Workflow Diagrams Each major system explained
🔐
Authentication Flow
Firebase Auth + JWT + DLT OTP
1
User taps Login — chooses Google, Email or Phone OTP on Flutter login screen
2
Firebase Auth verifies credentials → returns JWT ID token to Flutter app
firebase_auth.currentUser.getIdToken()
3
POST /api/auth/sync — Flutter sends token, FastAPI verifies with PyJWT, creates/updates user in MongoDB
4
JWT stored in flutter_secure_storage (Keystore/Keychain on mobile, localStorage on web)
5
OTP (DLT) — if phone auth: FastAPI calls MSG91 → DLT ledger validates → SMS delivered from YESHUB sender ID
Redis TTL: 600s · Max 3 attempts
🏠
Property Listing Flow
6-step wizard → publish
1
User taps Sell/Rent → 6-step form wizard opens on all platforms
2
Steps: Details → Location → Pricing → Media Upload → Contact → Review & Submit
3
Images uploaded to Firebase Storage → URLs stored in property document
image_picker → firebase_storage
4
POST /api/properties/ — saved to MongoDB with owner_uid, status: pending_review
5
Admin approves → is_verified: true → FCM push to owner → listing goes live
🚛
Packers & Movers Tracking
Live GPS + Uber-style assignment
1
Customer books — fills pickup/drop address, date, items. Razorpay collects 20–30% advance
2
MongoDB $geoNear finds nearest available mover → FCM sent to SP: "New job near you"
3
SP accepts (first-accepts-wins via Redis lock) → status: assigned → customer sees SP details on map
4
In Transit — SP app sends GPS every 5s via WebSocket → customer sees live truck + ETA
WS: /api/track/ws/{booking_id}/{role}
5
Delivery OTP confirms receipt → balance collected → Razorpay Route auto-splits to SP
💳
Payment Flow
Razorpay — UPI, cards, split payout
1
User taps Pay — Flutter calls POST /api/payments/create-order
Razorpay Orders API → order_id
2
Razorpay SDK opens — UPI/GPay/card/EMI options on mobile; JS checkout on web
3
Payment completes — Razorpay sends webhook to FastAPI
HMAC SHA256 signature verified
4
MongoDB updated — payment status: paid, booking status: confirmed
5
Razorpay Route auto-splits: platform commission + SP payout. FCM sent to both parties
✅
SP Verification (KYC) Flow
Aadhaar + PAN + Bank via Sandbox.co.in
1
SP uploads Aadhaar — file_picker opens doc picker → firebase_storage saves PDF to S3
2
Sandbox.co.in API called → OTP sent to Aadhaar-linked mobile via UIDAI
POST /kyc/aadhaar/otp · ₹5/call
3
SP enters OTP (pinput widget) → FastAPI verifies → is_aadhaar_verified: true in sp_profiles
4
PAN + Bank also verified via Sandbox + Razorpay KYC route
5
Verified badge shown on SP profile. RERA checked via APScheduler cron daily
💬
Real-time Chat Flow
WebSocket + Redis Pub/Sub + FCM
1
User taps Message on property → POST /api/chat/conversations creates/gets thread
2
WebSocket connects — wss://api.yestatehub.com/api/chat/ws/{uid}
web_socket_channel — works web + mobile
3
Message sent → FastAPI saves to messages collection → Redis Pub/Sub broadcasts to all server instances
4
Recipient receives live if connected, OR Firebase FCM push if app is closed
5
Read receipts — is_read: true + timestamp. Unread count badge updates on conversation list
